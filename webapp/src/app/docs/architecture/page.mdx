# Architecture

This project uses:

- A data pipeline running in [**Lambdas**](https://aws.amazon.com/lambda/) orchestrated by [**Step Functions**](https://aws.amazon.com/step-functions/) on [**AWS**](https://aws.amazon.com/)
- An [**OpenSearch**](https://opensearch.org/) cluster running on [**bonsai.io**](https://bonsai.io), for storing and searching the data.
- [**Gemini**](https://ai.google.dev/gemini-api) 2.5 Flash and Flash-Lite from Google AI, for extracting data from the CAC decisions.
- [**Next.js**](https://nextjs.org/) hosted on [**Vercel**](https://vercel.com/), for the web application.

It relies on [**Terraform**](https://www.terraform.io/) for cloud infrastructure management, and [**BAML**](https://boundaryml.com/) for type-safe LLM prompting and structured data extraction. The scraping is done using [**scrapy**](https://scrapy.org/). The frontend is built using [**Tailwind CSS**](https://tailwindcss.com/) and [**shadcn/ui**](https://ui.shadcn.com/) components.

### Company disambiguation

The pipeline uses the [**Companies House API**](https://developer.companieshouse.gov.uk/) to find companies house entries for the companies in the decisions.

It disambiguates entries based on the LLM's reasoning on the following:

- The company name
- The union (ie whether this union would be likely to seek recognition in this industry)
- The industrial classifications (nature of business) of the company
- The location of the bargaining unit
- The description of the bargaining unit (which sometimes contains clues as to the type of workers).

If the LLM believes based on its own knowledge that it should search for a different company name in Companies House, it is allowed to do this _once_.

The LLM is also allowed to identify an organisation as being a charity, a university, a central government department, or a local authority.
